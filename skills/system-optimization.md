# System Optimization — The Teacher

> **วัตถุประสงค์:** บันทึกบทเรียนจากงานที่มี Refinement เพื่อป้องกันความผิดพลาดซ้ำ  
> **ใครใช้:** `@po-agent` เรียกใช้หลังจบงานที่แก้ไขมากกว่า 1 รอบ  
> **อัพเดทล่าสุด:** เมษายน 2026

---

## เมื่อไหร่ต้องทำ

หลังจบงานที่มีการ **Refinement (แก้ไขมากกว่า 1 รอบ)** → `@po-agent` ต้องถาม User:

```
┌──────────────────────────────────────────────────────────────────┐
│ งานนี้เราแก้ไขหลายรอบ — มีบทเรียนที่ควรบันทึกไว้ครับ           │
│                                                                  │
│ ควรบันทึกลงที่ไหนดีครับ?                                        │
│                                                                  │
│ A) .cursorrules — ถ้าเป็น "กฎที่ทุก AI ต้องรู้" เช่น           │
│    ห้ามทำ X, ต้องทำ Y เสมอ                                      │
│                                                                  │
│ B) .cursor/skills/[ชื่อไฟล์].md — ถ้าเป็น "เทคนิคเฉพาะ" เช่น  │
│    วิธีจัดการ Layout แบบนี้, Pattern การ handle error แบบนี้    │
│                                                                  │
│ C) docs/codebase-docs/*.html + AI-GUIDE.md — ถ้าเกี่ยวกับโครงสร้างโปรเจค │
│                                                                  │
│ D) ไม่ต้องบันทึก                                                 │
└──────────────────────────────────────────────────────────────────┘
```

---

## ขั้นตอนหลัง User เลือก

เมื่อ User เลือก A, B, หรือ C → **`@po-agent` ลงมือแก้ไขไฟล์ทันที**

```
1. สรุปบทเรียนเป็น Rule/Guideline ที่กระชับ
2. เพิ่มลงในไฟล์ที่ User เลือก
3. แจ้ง User ว่าบันทึกแล้ว พร้อมแสดง content ที่เพิ่ม
```
