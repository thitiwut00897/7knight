## อ่านก่อนตอบ (ใน repo นี้เท่านั้น)

1. @docs/codebase-docs/.scan/PROJECT-CONTEXT.md — ข้อมูลสแกนจากโค้ด
2. @docs/codebase-docs/_template/HTML-TEMPLATE-GUIDE.md — รูปแบบ HTML ที่ต้องใช้ใน Phase 2
3. @docs/codebase-docs/_template/page-root.html และ page-feature.html — โครงหน้า
4. ถ้ามีไฟล์ .html อยู่แล้วใน docs/codebase-docs/ ให้เปิดอ่านโครงสร้างจริง (sidebar, section, class) เป็นตัวอย่าง

ห้ามอ้างอิงโปรเจกต์หรือ repo อื่นนอก workspace

## Phase 1 เท่านั้น — อย่าสร้าง/แก้ไฟล์ HTML ยัง

ตอบเป็น markdown ให้ตรวจสอบก่อน:

1. ฟีเจอร์หลัก (จัดกลุ่มตาม business) — หน้าที่, containers/screens
2. โครงสร้างโฟลเดอร์สำคัญ — data flow (UI → state → API)
3. สารบัญ HTML Phase 2 — ชื่อไฟล์, path, หัวข้อหลักในแต่ละหน้า (ให้สอดคล้องกับ template: index, overview, architecture, navigation, state-management, api-layer, components, theme-styling, utilities, features/*.html)
4. รายการ sidebar nav ที่จะใช้ร่วมกันทุกหน้า (ชื่อแสดง + href)

หลังตอบ ให้ user บันทึกเป็น docs/codebase-docs/OUTLINE-PHASE1.md ก่อนทำ Phase 2
